### 1. Чем final отличается от const в Dart?

| final | const |
|-------|-------|
| Значение определяется во время выполнения | Значение известно на этапе компиляции |
| Присваивается один раз | Полностью неизменяемая константа |
| Не может быть изменено после присвоения | Не может быть изменено после присвоения |
| `final now = DateTime.now();` | `const pi = 3.14159;` |
| Значение вычисляется при запуске программы | Значение вычисляется до запуска программы |

### 2. Что означает String? (с вопросительным знаком)

`String?` означает **nullable тип** — переменная может содержать либо строку (`String`), либо `null`.

```dart
String name = 'Иван';     // не может быть null
String? name2 = null;     // может быть null

### 3. Чем Future отличается от обычного значения? Что означает await с точки зрения потока выполнения?

**Future vs обычное значение:**

| Обычное значение | Future |
|-----------------|--------|
| Доступно сразу | Будет доступно позже |
| Синхронное | Асинхронное |
| Не требует ожидания | Требует await для получения значения |

```dart
// Обычное значение - доступно сразу
String name = 'Иван';
print(name);

// Future - будет доступно позже
Future<String> fetchName() async {
  await Future.delayed(Duration(seconds: 1));
  return 'Иван';
}

// Получение значения из Future
String name2 = await fetchName();

### 4. Зачем в Dart именованные конструкторы?
В Dart нет перегрузки конструкторов. Именованные конструкторы позволяют создавать несколько способов инициализации объектов с понятными именами:
```
class Point {
  double x, y;
  Point(this.x, this.y);
  Point.origin() : x = 0, y = 0;
  Point.fromJson(Map<String, double> json) 
    : x = json['x']!, y = json['y']!;
}
```
